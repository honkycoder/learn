---上午--- 1.JS的组成: DOM: 文档对象模型 BOM: 浏览器对象模型 ECMAScript: 规定了JS的核心语法 2.基本数据类型和复杂数据类型 基本数据类型: string number boolean undefined null 复杂数据类型: Object Date Math Array Function String Number Boolean 如何判断一个数据的类型: typeof 语法: typeof 数据(变量) 返回值类型: 字符串 注意点: 如果是复杂数据类型使用typeof是object,但是函数除外,函数是function 如何判断对象是否是一个数组: Array.isArray(对象) 3.等于和全等于 = 赋值: 把右边存储的信息复制一份给左边 == 等于: 判断右边数据是否与左边的数据相等 如果值类型和值类型进行判断,判断的就是具体值 如果是引用类型和引用类型进行判断,判断的是指向具体数据的地址 如果值类型和引用类型进行判断 把引用类型进行隐式转换,取出具体的值,再与值类型进行判断 === 全等于: 除了判断判断右边数据是否与左边的数据相等之外还会去判断类型 4.关系运算符: > < >= <= 5.逻辑运算符 逻辑非: 取反 语法: !表达式 结果: 是布尔类型 逻辑或 语法: 表达式1 || 表达式2 结果: 不是布尔类型 如果表达式1为真,返回表达式1 如果表达式1为假,返回表达式2 逻辑与 语法: 表达式1 && 表达式2 结果: 也不是布尔类型 如果表达式1为真,返回表达式2 如果表达式1为假,返回表达式1 6.值类型和引用类型 值类型存储的是具体的数据 引用类型存储的是指向具体数据的内存地址 7.值类型和引用类型赋值 赋值(=) : 把右边存储的信息复制一份给左边 值类型赋值: 把右边存储的信息(具体的数据)复制一份给左边 特点: 值类型赋值后,互不影响 引用类型赋值: 把右边存储的信息(指向具体数据的内存地址)复制一份给左边 特点: 引用类型赋值后,会共享同一份数据,修改其中一个对象的属性,会影响另外一个对象 8.值类型和引用类型在函数中的使用 实参: 实际的参数,函数调用时传入的参数 形参: 占位用的,函数调用前没有值 函数调用: 默认情况下会把实参的值赋值给形参 值类型作为函数的参数(值传递): 实参和形参是独立的,互不影响 引用类型作为函数的参数(地址传递): 实参和形参共享同一份数据,互相有影响 9.对象的动态特性 一个对象在创建出来之后,可以执行的操作: 添加 | 删除 | 修改 语法: 1.点语法 2.[]语法: []中必须是字符串 注意点: 1.如果给一个对象的属性赋值,如果这个对象有这个属性就是修改,如果没有这个属性就是添加 2.使用点语法很方便,但是有时候使用点语法无法来获取对象的属性(属性有空格 | 变量的值是字符串只能使用[]语法) 10.in关键字 1.for..in遍历对象(key-value键值对)中的属性 2.判断对象是否有指定的属性 语法: 属性 in 对象 注意点: 属性必须是字符串 返回值: 布尔类型 注意点: 1.操作数组的时候不是操作数据的元素,而是操作数据的下标 2.判断数组中是否有某个元素? 1.遍历整个数组,获取每个元素判断对应的值 2.indexOf -> 元素在数组中的下标,如果没有 -> -1 11.delete 1.删除对象中的属性 2.删除未使用var声明的变量 注意点: 1.这个结果是有返回值,类型为布尔类型,true代表删除成功,false代表删除失败 2.不能删除使用var声明的变量 3.但是可以删除直接定义在window上的属性 ---下午--- 1.调试工具的使用 如何打开? F12(Windows) command + option + j/i(MAC) console和js运行界面一致 JS中的断点: 普通断点 + 条件断点(达到一定条件才会触发这个断点,让程序停止) 2.循环和分支 循环: for | for..in | while: 先判断再执行,有可能一次都不执行 | do..while: 先执行后判断,至少执行一次 3.异常处理 正在运行代码如果出错了或者有异常,后面的代码就无法执行了,但是有时候代码就算出现错误或者异常,后面的代码依旧想要执行,可以使用异常捕获结构 异常捕获完整结构: try { // 可能出错的代码 } catch (e) { // 如果上面的代码出错了,就会来到这个代码块 // e: 错误信息 } finally { // 不管上面的代码有没有出错,都会执行 // 用于释放一些资源,前端有垃圾回收机制,不需要去管理内存,后端才需要去关心(Node.js) } 手动抛出异常: throw 错误信息(字符串 | 对象) 4.DOM操作: 对界面上的元素进行增删改查 5.函数和对象的创建 函数的创建方式 1.函数声明 function 函数名(){} 2.函数表达式 匿名函数表达式 var sum = function (){} 命名函数表达式 var add = function fn(){} 3.Function构造函数 var func = new Function();// var func = function (){} var func = new Function('console.log(\'demo\')'); 对象的创建方式 1.字面量方式 var obj = {}; 2.构造函数方式 var obj = new Object(); // var obj = {}; 3.如果是创建的Date,是当前时间 var date = new Date(); 6.面向对象和面向过程编程: 它们都是解决问题的思路(思想) 区别: 解决问题的关注点不同 面向过程关注的是解决问题的每一步步骤 面向对象关注的是解决问题所需要的对象 没有孰优孰劣之分,只是解决问题的思路不一样 7.面向对象编程的相关概念. 1.对象 什么是对象? 万物皆对象,人车狗 一般说的对象是具体的事物 2.对象的静态描述信息 --> 对象中的属性(对象中创建的变量) 3.对象的动态行为特征 --> 对象中的方法(对象中创建的函数) (张三)牵着一条叼着(骨头)的(狗)在等(公交车); 8.为什么要使用面向对象编程 1.计算机的存在是为了帮助人们解决问题,使用面向对象编程可以更好的去解决问题 2.使用面向对象,代码的结构性更好,可以更好的去描述一个客体(可以想象到的或者感知到的任何事物) 3.维护性更好,复用性更好

1.调试工具的使用 如何打开? F12(Windows) command + option + j/i console和js运行界面一致 JS中的断点: 普通断点 + 条件断点(达到一定条件才会触发这个断点,让程序停止) 2.循环和分支 循环: for | for..in | while: 先判断再执行,有可能一次都不执行 | do..while: 先执行后判断,至少执行一次 3.异常处理 正在运行代码如果出错了或者有异常,后面的代码就无法执行了,但是有时候代码就算出现错误或者异常,后面的代码依旧想要执行,可以使用异常捕获结构 异常捕获完整结构: try { // 可能出错的代码 } catch (e) { // 如果上面的代码出错了,就会来到这个代码块 // e: 错误信息 } finally { // 不管上面的代码有没有出错,都会执行 // 用于释放一些资源,前端有垃圾回收机制,不需要去管理内存,后端才需要去关心(Node.js) } 手动抛出异常: throw 错误信息(字符串 | 对象) 4.DOM操作: 对界面上的元素进行增删改查 5.函数和对象的创建 函数的创建方式 1.函数声明 function 函数名(){} 2.函数表达式 匿名函数表达式 var sum = function (){} 命名函数表达式 var add = function fn(){} 3.Function构造函数 var func = new Function();// var func = function (){} var func = new Function('console.log(\'demo\')'); 对象的创建方式 1.字面量方式 var obj = {}; 2.构造函数方式 var obj = new Object(); // var obj = {}; 3.如果是创建的Date,是当前时间 var date = new Date(); 6.面向对象和面向过程编程: 它们都是解决问题的思路(思想) 区别: 解决问题的关注点不同 面向过程关注的是解决问题的每一步步骤 面向对象关注的是解决问题所需要的对象 没有孰优孰劣之分,只是解决问题的思路不一样 7.面向对象编程的相关概念. 1.对象 什么是对象? 万物皆对象,人车狗 一般说的对象是具体的事物 2.对象的静态描述信息 --> 对象中的属性(对象中创建的变量) 3.对象的动态行为特征 --> 对象中的方法(对象中创建的函数) (张三)牵着一条叼着(骨头)的(狗)在等(公交车); 8.为什么要使用面向对象编程 1.计算机的存在是为了帮助人们解决问题,使用面向对象编程可以更好的去解决问题 2.使用面向对象,代码的结构性更好,可以更好的去描述一个客体(可以想象到的或者感知到的任何事物) 3.维护性更好,复用性更好

异常处理: 正在运行的代码出错了或者发生异常,后面的代码就不会执行了,有时候我们希望就算代码出错了或者发生异常,后面的代码还能执行,这个时候可以使用异常捕获结构 异常捕获结构格式: try { // 可能会出错的代码 } catch (e) { // 如果上面的代码出错了,就会来到这个代码块 // e: 错误信息 } 手动抛出异常: throw 错误信息(字符串 | 对象) 公司规定: (msg + code) 1001 变量未定义 异常捕获完整结构 try { // 可能会出错的代码 } catch (e) { // 如果上面的代码出错了,就会来到这个代码块 // e: 错误信息 } finally { // 不管上面的代码有没有出错都会来到这个代码块 // 为了释放一些资源,但是前端开发有垃圾回收机制,所以一般不会使用这个,后端开发的时候才会使用(Node.js) }
